# Exercise 3 - Sorting Customer Orders

## Objective
Sort customer orders based on total price.

## Algorithms Used
- Bubble Sort
- Quick Sort

## Time Complexity

### Bubble Sort
- Best: O(n)
- Average: O(n²)
- Worst: O(n²)

### Quick Sort
- Best: O(n log n)
- Average: O(n log n)
- Worst: O(n²) (rare)

## Conclusion
Quick Sort is preferred for large datasets because it is much faster on average.
